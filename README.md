# Информационная система мониторинга криптовалютного портфеля

MVP дипломного проекта для учета и мониторинга криптовалютного портфеля:
- PostgreSQL с процедурами и триггерами для бизнес-логики;
- FastAPI backend;
- Telegram-бот на aiogram 3.x;
- Telegram WebApp с русским интерфейсом;
- асинхронный клиент CoinGecko с получением цен в рублях;
- поддержка BTC, ETH, SOL, BNB, XRP, DOGE, ADA, TON, TRX и LINK;
- Docker Compose для локального запуска.

## 1. Подготовка

1. Скопировать переменные окружения:
   - Linux/macOS: `cp .env.example .env`
   - Windows PowerShell: `Copy-Item .env.example .env`
2. Заполнить `BOT_TOKEN` в `.env`.
3. Заполнить `WEBAPP_URL` публичной HTTPS-ссылкой на backend, например `https://your-domain/webapp/`.
4. Заполнить `SITE_URL` публичной HTTPS-ссылкой на альтернативный сайт, например `https://your-domain/site/`.
5. Если нужен прокси для внешних API, заполнить `PROXY_HOST`, `PROXY_PORT`, `PROXY_USERNAME`, `PROXY_PASSWORD` или `PROXY_FALLBACKS`.

## 2. Запуск через Docker Compose

```bash
docker compose up --build
```

Сервисы:
- `postgres` - база данных и автоинициализация схемы из `db/001_init_schema.sql`;
- `backend` - FastAPI (`http://localhost:8010`, WebApp на `/webapp/`, сайт на `/site/`);
- `bot` - Telegram-бот с кнопкой открытия WebApp и отправкой ценовых уведомлений.

При полной замене схемы нужно пересоздать локальный volume PostgreSQL, иначе старая база может содержать прежние имена колонок.

## 3. Быстрая проверка API

- `GET /health` - проверка доступности сервера;
- `POST /users/register` - регистрация пользователя;
- `POST /portfolios` - создание портфеля;
- `POST /operations` - добавление покупки или продажи;
- `GET /portfolios/{portfolio_id}/balance` - получение баланса портфеля;
- `POST /market/sync` - синхронизация рублевых котировок;
- `POST /app/auth` - авторизация пользователя WebApp по данным Telegram;
- `GET /app/dashboard` - сводка главной вкладки WebApp;
- `GET /app/market` - карточки рынка с логотипами и изменением цены за 24 часа;
- `GET /app/prices` - текущие цены активов;
- `POST /app/trade` - операция из WebApp;
- `GET /app/alerts` - список ценовых уведомлений;
- `POST /app/alerts` - создание ценового уведомления;
- `POST /app/alerts/{alert_id}/disable` - отключение уведомления.

Swagger доступен по адресу `http://localhost:8010/docs`.

## 4. Telegram WebApp

После команды `/start` бот отправляет кнопку `Открыть приложение`.

В WebApp доступны разделы:
- `Главная` - общая стоимость портфеля, метрики, витрина рынка, позиции и быстрые действия;
- `Операция` - покупка или продажа актива по рыночной или ручной цене;
- `Рынок` - карточки поддерживаемых активов с логотипами, ценой, рангом и изменением за 24 часа;
- `Алерты` - создание, список и отключение ценовых уведомлений.

В форме продажи доступны быстрые клише на 25%, 50%, 75% и 100% от текущей позиции.

Для запуска внутри Telegram нужен HTTPS URL и настройка у `@BotFather`:
- `/mybots` -> выбрать бота -> `Bot Settings` -> `Menu Button`;
- указать `WEBAPP_URL`, например `https://your-domain/webapp/`.

## 5. Команды Telegram-бота

- `/start` - регистрация пользователя и кнопка открытия WebApp;
- `/menu` - повторно показать кнопку открытия WebApp;
- `/site` - персональная ссылка на альтернативный вход через обычный браузер;
- `/balance` - баланс основного портфеля;
- сообщение вида `1 btc` - быстрый расчет стоимости актива в рублях.

## 6. Автопроверка цен и уведомления

Бот запускает фоновый цикл, который:
- периодически получает цены с CoinGecko в рублях;
- вставляет новые значения в `market_data`;
- триггеры PostgreSQL проверяют `price_alert`;
- при срабатывании бот отправляет пользователю уведомление в Telegram.

## 7. Бизнес-логика в базе данных

В `db/001_init_schema.sql` реализовано:
- фиксированная комиссия `0,7%` от сделки;
- пересчет `portfolio.total_value_rub` триггерами при вставке котировки или операции;
- расчет средней цены покупки и реализованного результата в рублях;
- проверка и срабатывание ценовых уведомлений.
